教你一个**国内最有效、最简单的“暴力”破网法**。

这个报错的本质是：你电脑的操作系统不知道 GitHub 服务器的真实 IP 地址，去问国内的 DNS 服务器时，被给了一个“错误的或者被拦截的”地址。

我们直接在本地电脑的 `hosts` 文件里把 GitHub 的真实地址写死，这样 Git 就能直接找到路了。

### 🛠️ 一分钟配置 Hosts 解决网络拦截

#### 第一步：以管理员身份打开“记事本”

1. 点击 Windows 电脑左下角的 **开始菜单（或按 Win 键）**。
2. 在搜索框里输入 **“记事本”**。
3. 在搜索结果的“记事本”上**右键**，选择 **“以管理员身份运行”**（这一步非常重要，否则后面无法保存）。

#### 第二步：打开 hosts 文件

1. 在记事本顶部菜单点击 **文件 -> 打开**。

2. 在弹出的文件路径框里，直接复制并粘贴下面这个路径，然后敲回车：

   Plaintext

   

   ```
   C:\Windows\System32\drivers\etc
   ```

3. 此时你会发现文件夹里是空的，别慌！把右下角的 **“文本文件(\*.txt)”** 切换成 **“所有文件(.)”**。

4. 双击打开看到那个名叫 **hosts** 的文件。

#### 第三步：把 GitHub 的高速公路地址贴进去

滚动到记事本的最底部，另起一行，把下面这几行目前国内最新、最稳定的 GitHub 官方 IP 地址复制并粘贴进去：

Plaintext



```
# GitHub Fast Start
140.82.112.4 github.com
199.232.69.194 github.global.ssl.fastly.net
185.199.108.153 assets-cdn.github.com
185.199.109.153 assets-cdn.github.com
185.199.110.153 assets-cdn.github.com
185.199.111.153 assets-cdn.github.com
```

1. 粘贴好后，按下键盘组合键 `Ctrl + S` 保存文件，然后关闭记事本。

### 🚀 重新验证

配置完成后，我们不需要重启电脑。回到你的 VS Code / PowerShell 终端里。

1. 先刷新一下系统的网络缓存（直接运行这行系统命令）：

   PowerShell

   

   ```
   ipconfig /flushdns
   ```

2. 重新发起拉取（因为云端有更新，必须先拉取）：

   Bash

   

   ```
   git pull origin main
   ```

3. 只要拉取成功了，立刻趁热打铁推送：

   Bash

   

   ```
   git push origin main
   ```

这次直接走死地址，通常就能绕过拦截，直接秒传成功了！快去试试！
